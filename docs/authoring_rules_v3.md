## v2での主な変更点（要点・継続）
1. **解説の太字ルールを明文化**（§5）。問題文＝強調禁止、解説＝強調推奨、という役割の非対称を確定。

4. **画像・版権の指針を明文化**（§10）。オリジナルSVG主軸。外部画像は人間がライセンス確認したものに限る。
5. **作問エージェントの最終出力形式を明文化**（§11）。検証済み1問JSON（スキーマv2準拠）＋自己チェックで出力。

## 1. 選択肢（最重要）

- **★ 選択肢は「行為のみ」をシンプルに記載する。**「〜を疑い」「〜と判断し」「〜と考え」「〜として」「〜を最優先し」などの**診断的・根拠的な修飾語は一切付けない。**
  - ✗ 悪い例：「悪性腫瘍に伴う高カルシウム血症を疑い、腫瘍検索を最優先する」
  - ◯ 良い例：「悪性腫瘍を検索する」
  - ✗ 悪い例：「散発性の原発性副甲状腺機能亢進症と確定し、副甲状腺摘出術を急ぐ」
  - ◯ 良い例：「副甲状腺を摘出する」
- 正答を**最長にしない**（長さブラフの回避）。選択肢は**同カテゴリ・同程度の長さ・同程度の妥当性**で揃える。
- **1問1論点**に絞る。
- 消極的・安全な選択肢（「経過観察」「再度病歴聴取」「診断を再考」）を**正答にしない**。正答は具体的アクションにし、"同じくらい安全に見えるが誤り"の保守的ディストラクターを添える。
- 藁人形（明らかに不合理）なディストラクターを作らない。各誤答は実際に起こりうる認知エラーを反映させる。

## 2. 問題文（stem）

- **★ 病歴は「現病歴 → 既往歴・内服歴」の順**。診断を露骨に指し示す情報（例：原因薬剤の長期内服）を**冒頭に置かない**（telegraph回避）。鍵となる薬剤・既往は、**他の併存疾患・併用薬の中にさらっと混ぜて**目立たせない。
- **★ 鍵となる検査値（例：薬物血中濃度）は、検査項目の最後に置く。**
- **★ 問題文（stem）に太字などの強調を付けない。**強調は強烈なヒント（telegraph）になり識別を損なう。重要所見も平文で提示し、判断は受験者に委ねる（**強調は解説側でのみ用いる → §5**）。
- 検査結果に**解釈タグを付けない**（「（高値）」「（陽性）」「（非抑制）」「〜が示唆される」等は不可）。客観値・参照範囲・定性結果のみを提示し、解釈は受験者に委ねる。
- 設問は「**最も適切な対応／次の一手／管理**」を問う。診断名当ては避ける（古典的提示では識別が効かない）。
- 年齢・性別・主訴・現病歴・既往・内服・生活歴・バイタル・身体所見・検査・セッティングを十分に記載する。

## 3. 難易度

- 総合診療「**専門医**」が現場で差をつける点を狙う：**管理・合併症予測・検査解釈・治療のsequencing・禁忌・モニタリング・薬物相互作用**。
- 医師国家試験・初期研修レベルの単純想起問題にしない。古典的提示には競合する鑑別や subtle な管理点を足して難化させる。
- 推定正答率 p は 0.3〜0.8（平均6〜7割）に散らばるよう設計。難易度の自己審査は強化追補モジュール§Dのゲートを通す。

## 4. 引用・エビデンス

- **一次文献（PubMed）を優先**し、出版年の新しいものを優先する。総説・主要ガイドライン・メルクマールRCT。
- **全PMIDを独立に実在・内容照合**（According to PubMed）。HR/OR/CI・割合は**原著抄録で確認した値のみ**記載。数値は記憶で書かず、出典本文から取る。
- PubMed未収載の国内ガイドライン細目・新版規定・添付文書事項は「**要確認**」と明示する。

## 5. 解説

### 5-1. 構成
- 順序：**正解根拠 → 誤答ごとの分析（＋認知バイアス名）→ 主要エビデンス表（PMID）→ 最新ガイドライン（箇条書き）→ 習得すべき要点 → 参考文献（Vancouver番号付き）**。
- 「1問のテーマから波及して幅広く深く学べる」密度。
- 正解根拠には、**症例所見 → 該当する分岐枝 → 数値/正解**の導出を必ず一文で明記する（強化追補モジュール§C-2）。
- 【最新ガイドラインと主要エビデンス】は**箇条書きで改行**して可読性を確保。
- **習得すべき要点（points）は4〜5個を必須**。空にしない（強化追補モジュール§F）。

### 5-2. ★ 太字（強調）ルール ―― 問題文とは正反対
- **解説では、重要部分を太字で強調する。** 対象＝結論・鍵となる数値・決定的な判断・禁忌・正答の根拠・鑑別の決め手。読み手が要点を一目で把握できるようにする。
- これは§2の問題文（stem）の**「太字禁止」とは正反対**の扱いである。理由は役割の違い：**問題文＝受験者に判断を委ねる場**（強調はtelegraphになるため禁止）、**解説＝学ばせる場**（強調で理解を導くため推奨）。
- ただし**乱用しない**。全部太字は強調にならない。1文・1段落あたり、本当に効く要点に絞る。
- 見本＝規範問題の解説（結論・数値・禁忌が的確に太字）。反面教師＝太字ゼロで全文が同じ濃さの解説（読み手が肝を自力で探す羽目になる）。**これは今後は避ける。**

### 5-3. ★ PICOタブ（主要エビデンスのRCT展開）仕様
- **主要エビデンスで挙げたRCT等の介入研究には、PICOタブ（クリックで展開）を必ず付ける。**
- タブの中身：
  - **P / I / C / O**：O（アウトカム）は**絶対値・HR/OR・95%CIを含める**。
  - **NNT**：提示可能なら記載。**ARR等の算出根拠を併記**する。
  - **批判的吟味（limitation＝caveat）**：その研究の弱点・適用限界を一文で。例＝企業資金源、対象集団の限定、短期評価、早期中止（症例数不足）、サロゲートアウトカム、非劣性マージンの妥当性 等。
- **付与条件**：`rct: true`（介入研究）なら**必須**。ガイドライン・総説・観察研究・メタ解析（`rct: false`）には**付けない**（無理に付けない）。
- データ構造：スキーマv2の `evidence[].pico`（P/I/C/O/NNT/caveat）に格納。レンダラーが `pico` の有無でタブを描画する（データがあれば出る、無ければ出ない）。
- **規範例（この密度・書式を標準とする）**：P/I/C/Oが2×2で揃い、緑のNNTバッジ（値＋ARR/追跡年）、オレンジの批判的吟味バッジ（資金源・反応原性・忍容性等）が付いた展開タブ。

### 5-4. 図・表
- 解説には、理解を助ける図・表を積極的に入れる（**判断基準・品質基準は §10**）。

## 6. 体裁・その他

- 出力は共通レンダラーが描画する（データ駆動）。解説内のHTML（表・強調・図）はレンダラーの仕様に沿う。
- 薬は**一般名（商品名®）**。ワクチンは**商品名®を併記**。
- 解答分布はA〜Eで偏らせない。正答位置・長さにパターンを作らない。
- **★ 成果物に氏名を一切入れない。**

## 7. QA（検証）

- **独立解答法**：正答キーを伏せた別の解き手に解かせ、(1)一義に決まるか (2)割れないか (3)想定外の正解が成立しないか を点検。割れたら修正。
- **難易度ゲート**（強化追補モジュール§D）：中堅受験者が鍵知識なしに2択以上消せないか／想起で解けないか／推定正答率 0.3〜0.8 か。
- 解答分布・選択肢パース・HTML構造・**PICOタブの表示**（rct:true に pico が揃い展開するか）・**太字の付与**・**図表の適否**を機械的＋目視で確認。
- 高stakesな検証は独立サブエージェントに回す（事実確認／品質・弁別。詳細は別文書「作問レビューエージェント設計」）。

## 8. 運用（テーマ管理・カテゴリー）

### 8-1. カテゴリー基準
- **★ 領域＝「カテゴリー分類 確定仕様 v2」の24カテゴリー**を用いる。テーマ選定・重複回避・カバレッジ管理は、この24カテゴリー（および各問1つの `category`）を軸に行う。未出題・手薄なカテゴリーを優先する。
- 24カテゴリーの構成：A群（imCore=true・16／総合内科モードでも表示）＝総合内科・臨床推論／循環器／呼吸器／消化器・肝胆膵／腎臓・電解質／内分泌・代謝／血液／神経／膠原病・アレルギー／感染症／救急・集中治療／老年医学／緩和ケア／EBM・臨床研究／運動器・整形外科／精神・行動医学。B群（imCore=false・8／総合診療モードのみ）＝小児・思春期／女性・男性医療（周産期含む）／予防医療・公衆衛生／地域医療・医療制度・在宅／皮膚科／眼科／耳鼻咽喉科／泌尿器科。
- `category` の値は上記24名称のいずれかと一字一句一致させる（詳細はカテゴリー分類 確定仕様 v2）。
- ユーザーが「確定」と言ったら、テーマ管理を最終テーマで更新する。反映項目：**category**・トピック・出題の軸・想定正答（A〜E）・状態（=確定）・重複キー・出典。
- 確定後はテーマ案と実際の作問内容が乖離していないか（category・トピック）を突き合わせる。

### 8-2. ★ マイナー科（皮膚・眼・耳鼻咽喉・泌尿器）の出題範囲の限定
- これら4科の問題は、**専門医が自ら扱う深さではなく、家庭医療専門医／総合診療医が「関わる範囲」に限定する。** すなわち、プライマリ・ケアで遭遇し、**初期対応・鑑別・重症度判断・専門医紹介の判断**ができる深さまで。専門科の高度手技・稀な病理分類・専門治療の細目は対象外。JPCA「日常病」リストに挙がる疾患群が、この"家庭医が診る範囲"の目安。
- **皮膚科**：接触性/アトピー性皮膚炎、蕁麻疹、ざ瘡、白癬・カンジダ、疥癬、膿痂疹、軟属腫、粉瘤、褥瘡、皮脂欠乏性皮膚炎、（救急的な）薬疹・蜂窩織炎・帯状疱疹の初期対応と紹介判断。※皮膚生検・腫瘍の病理診断・専門的光線療法は範囲外。
- **眼科**：結膜炎、麦粒腫、白内障・緑内障の気づきと紹介、糖尿病網膜症のスクリーニング・紹介、急性視力障害・眼痛のレッドフラッグと救急紹介。※眼底精査・眼科手術・専門的眼圧管理は範囲外。
- **耳鼻咽喉科**：アレルギー性鼻炎、中耳炎、副鼻腔炎、BPPV（Epley等）、扁桃周囲膿瘍・急性喉頭蓋炎のレッドフラッグと救急紹介、鼻出血の初期対応。※内視鏡下手術・専門的聴力精査・頭頸部腫瘍治療は範囲外。
- **泌尿器科**：尿路感染（膀胱炎・腎盂腎炎）、過活動膀胱、前立腺肥大症、勃起障害、尿路結石の初期対応と紹介、排尿障害・尿失禁・血尿の鑑別とレッドフラッグ、（男性医療として）性の健康。※尿路内視鏡・泌尿器手術・専門的がん治療は範囲外。
- 作問時は、各設問の主眼が「家庭医の初期対応・鑑別・紹介判断」に収まっているかを確認する。専門医レベルの細目に踏み込んでいたら難易度・範囲を再調整する。

## 9. 作問ワークフロー（★標準手順）

ユーザーが「次の5問を作成」等と指示したら、以下を順に実行する。

1. **現在の日時を確認**し、最新の年・版で推論する。ハルシネーションを避け、不明は「わからない／要確認」と明言。
2. **接地テーマ抽出**（強化追補モジュール§A）：大テーマ→最新GL＋メルクマールRCT収集→小テーマ3〜5案（分岐点／罠／文献PMID／高yield理由付き）。カテゴリー分類v2（24カテゴリー）の未出題・手薄を優先。**独立サブエージェント⓪でテーマ吟味**（専門性・難易度の絶妙さ・推定正答率約70%。往復最大3回）。合格テーマのみユーザーへ提案。**最終選択は人間**。
3. **一次資料接地**（強化追補モジュール§B）：主要一次文献を精読し、用量・カットオフ・目標値・NNT/HR/CI、特に**条件分岐**を抽出。
4. **問題と解説を作成**：few-shot錨（規範問題2〜3問）を添付し、暗黙知3則（§C）を厳守。各問＝問題文（強調なし・telegraph禁止）／行為のみ5選択肢／正答／完全解説（§5：正解根拠→誤答分析＋バイアス名→主要エビデンス表〔全PMID独立照合・RCTにPICOタブ〕→最新ガイドライン→習得すべき要点4〜5→参考文献→適切な図表→**解説は太字で要点強調**）。
5. **独立サブエージェント①**が「問題のみ（解説・正答キーなし）」を読み、批判的に吟味（藁人形の有無・識別能・独立解答法）。
6. **独立サブエージェント②**が解説の情報の正誤（数値・PMID・GL・**PICOの値**）と深さ・網羅性を検証。
7. **フィードバックに則り調整**。往復は**最大3回**。停止は収束で決める（詳細は「作問レビューエージェント設計」）。

- 完成後：**§11の最終出力形式**で成果物を確定 → data.js へ統合（新 setId／単一 category）→ 構文（node --check）・解答分布・重複を機械検証 → テーマ管理更新 → 氏名なし。

## 10. 図・表・画像の標準手順（★グランドルール）

### 10-1. 図・表を「いつ」入れるか（判断基準）
- **★ 解説には、内容の理解を助ける図・表を積極的に入れる。ただし「図のための図」は禁止。** その図/表が、文章より明確に理解を助ける場合に**限る**（強化追補モジュール§F）。数を目標にしない（質が下がる）。効くかどうかを目標にする。
- **表が効く場面（推奨・多くの問題で有効）**：比較（A vs B）、条件分岐（症例→該当枝→数値）、鑑別の一覧、診断基準・除外基準の対照、用量・カットオフの整理。表は図より確実。
- **図（フロー／模式図）が効く場面**：制度・診断・治療の**流れ（フロー）**、解剖・機序の**空間関係**、アルゴリズム、ECG／画像所見の模式化。
- **入れない方がよい場面**：単一の数値・短い判断で完結する内容、文章で十分な内容。ここに無理に図を足さない。
- **★「よくある誤り」ボックス（推奨パターン）**：フロー図に「各誤答＋なぜ誤りか」を添えると、ディストラクター解説と図が連動して効果的。

### 10-2. 図の品質基準
- ★ **フォント**：SVG図の文字は **SF Pro（英数）＋ ヒラギノ角ゴ（日本語）**。
  例：`font-family="-apple-system, 'SF Pro Text', 'Hiragino Kaku Gothic ProN', 'Hiragino Sans', sans-serif"`
- ★ **必須の評価ループ**：図を作成したら、**プロの目で客観評価し、厳しく指摘して作り直す工程を最大3回**行ってから提示する。評価観点＝(1)医学的正確性 (2)情報の階層・可読性 (3)ラベルの明瞭さ・重なり・整列 (4)色の意味付けの一貫性 (5)余白・バランス。**3回を超える往復はしない。中途半端なら省略する**（中途半端な図はむしろ質を落とす）。
- 実描画で確認する（ブラウザ描画のスクリーンショットを自分の目で見る等）。

### 10-3. ★ 画像・版権の指針
- **原則：オリジナルSVG図を主軸とする。** 著作権問題がなく、テーマに最適化でき、サイトのデザインと揃う。**これが第一選択。**
- **外部画像（版権フリー／CCライセンス写真等）**：オリジナルSVGでは表現できない実物（特定の皮疹の見た目 等）に**限り**使用可。ただし——
  - **ライセンスを人間（作問者）が必ず確認する**（商用利用可・改変可・クレジット要否）。**AI／エージェントの「フリーだから使う」という自動判断に委ねない**（ライセンスは変動し、表示と実条件が食い違うことがある）。
  - **臨床写真は患者由来・プライバシーのリスク**があるため特に慎重に。出所不明の臨床写真は使わない。
- **論文の図表の直接転載は原則禁止**（出版社が著作権を持つ。オープンアクセスでも条件付き）。**データ（数値）を引用してオリジナル図に描き直すのはOK、図そのものの転載はNG。**

## 11. ★ 作問エージェントの最終出力形式

作問エージェント（Cowork 等）の**最終成果物は、次の形で出力する**。これにより、Cowork で作った成果物を Claude Code がそのまま `QUESTIONS` 配列へ追記でき、「作り方→データ形式→サイト反映」が一本化される。

### 11-1. 出力する本体：検証済み1問JSONオブジェクト（スキーマv2準拠）
- 1問＝1オブジェクト。フィールド：
  `category`（**確定仕様v2の24カテゴリーのいずれかと一字一句一致**）／`setId`／`title`／`meta`／`stem`／`options`（原則5・行為のみ）／`answer`（**0始まり**）／`explain`。
- `explain` ＝ `core`（正解根拠・導出明示・**太字で要点強調**）／`fig`（**§10の基準で入れる図表。不要なら空文字 `""`**）／`evidence[]`（`{src,pmid,point,rct}`。**`rct:true` なら `pico` 6項目〔P/I/C/O/NNT/caveat〕必須**）／`distractors`（各誤答＋認知バイアス名）／`guideline[]`／`points`（**4〜5個・空禁止**）／`refs`（Vancouver＋PMID）。
- 旧 `dom`／`domEn`／`cat` は使わない（スキーマv2）。

### 11-2. 併せて添える：自己チェック結果
成果物に、以下を通したことの自己申告を付ける（スキーマv2の検証チェックリスト準拠）：
- [ ] JSONとして妥当（`node --check` 相当）／必須フィールド完備
- [ ] `category` が24カテゴリーのいずれかと一字一句一致／旧フィールド不在。マイナー科は家庭医が関わる範囲に収まっている（§8-2）
- [ ] `answer` が整数かつ範囲内／`options` 原則5
- [ ] `evidence` の `rct:true` に `pico` 6項目完備／`points` 4〜5個で非空
- [ ] `pmid` 実在＋内容一致を確認（According to PubMed）／`refs` に空白詰まりなし
- [ ] `stem` に強調・解釈タグ・telegraphなし／`core` に §C の導出明示
- [ ] **解説の太字**が要点に付いている（乱用なし）／**図表**が §10 の基準（効く場面・品質）を満たす、または適切に省略
- [ ] 独立解答法・難易度ゲート（§7）を通過／解答分布が偏っていない

### 11-3. 出力の作法
- 氏名は一切入れない。
- レビュー（事実確認・品質弁別・回帰テスト）は「作問レビューエージェント設計」に従う。最終審級は**人間の目＋実データの識別指数**。
