## 0. 設計方針（確定）

- **1問 = 1カテゴリー**（単一分類）。従来の `dom`（主領域）＋ `cat`（複数タグ配列）の**二重体系は廃止**し、単一の分類軸に統一する。
- **全20カテゴリー**。順序は下表のとおり（表示順＝この順）。
- 各カテゴリーに **`imCore`（総合内科モードで表示するか）** のフラグを1つ持たせる。
- **総合内科モードの定義**：「**総合内科専門医試験でカバーされる範囲のみを表示する**」。この1本の定義で、トグルの絞り込み（除外）がすべて決まる。総合内科モードで隠れるのは下記4カテゴリー（imCore=false）のみ。
- カテゴリー名は、確定後は **`id` と同格の不変キー**として扱う（安易に改名しない。改名は表示とデータ両方の一括変更を要する）。

### 拠り所（一次情報）
- **内科側**：日本内科学会 J-OSLER の内科13分野／MKSAP 19 の章立て（循環器・内分泌代謝・消化器肝臓・総合内科・血液・感染症・腎臓・神経・腫瘍・呼吸器集中治療・リウマチ・皮膚科）／総合内科専門医試験の範囲。
- **総合診療側**：日本専門医機構 総合診療専門研修 整備基準（必須領域：内科・小児科・救急科。加えて外科・整形外科・産婦人科・精神科・皮膚科・眼科・耳鼻咽喉科。および地域包括ケア・在宅医療・医療福祉制度・医療経済）。
- ※これらは論文（PubMed一次文献）ではなく**カリキュラム／ブループリント**が一次情報である点に留意（この用途ではこれが一次情報）。

## 2. 実装メモ（推奨。詳細な変数名・段取りは Claude Code と現物で調整）

- 各問題に**単一フィールド**（例 `category`：文字列）を持たせ、値は上表の名称と**一字一句一致**させる。
  - 従来の `dom`（string）・`cat`（array）は廃止し `category` に集約する。※旧 `cat`（配列）を再利用すると型が変わり既存コード（`categories()` 等）が壊れるため、**新しいフィールド名で clean に移行**するのが安全。
- マスター定義を**1箇所**に置き、表示順とフラグの単一の真実とする：

```js
const CATEGORIES = [
  { name: "総合内科・臨床推論",   imCore: true  },
  { name: "循環器",              imCore: true  },
  { name: "呼吸器",              imCore: true  },
  { name: "消化器・肝胆膵",       imCore: true  },
  { name: "腎臓・電解質",         imCore: true  },
  { name: "内分泌・代謝",         imCore: true  },
  { name: "血液",                imCore: true  },
  { name: "神経",                imCore: true  },
  { name: "膠原病・アレルギー",   imCore: true  },
  { name: "感染症",              imCore: true  },
  { name: "救急・集中治療",       imCore: true  },
  { name: "老年医学",            imCore: true  },
  { name: "緩和ケア",            imCore: true  },
  { name: "EBM・臨床研究",       imCore: true  },
  { name: "運動器・整形外科",     imCore: true  },
  { name: "精神・行動医学",       imCore: true  },
  { name: "小児・思春期",         imCore: false },
  { name: "女性医療（産婦人科）", imCore: false },
  { name: "予防医療・公衆衛生",   imCore: false },
  { name: "地域医療・医療制度・在宅", imCore: false },
];
```

- **chip件数**：問題を `category` でグルーピングして数える（件数0のカテゴリーは非表示にしてよい）。
- **総合内科モードの絞り込み**（脆い文字列名指し除外を廃し、フラグ1つで判定）：
  - 表示カテゴリー ＝ `CATEGORIES.filter(c => c.imCore)`
  - 出題プール ＝ `scope === 'gp' ? Q : Q.filter(q => catIsImCore(q.category))`
    （`catIsImCore` は `CATEGORIES` を引いて `imCore` を返すだけ）

## 4. 要確認・注意

- **Q1「公衆衛生・EBM」**：問題内容がEBM/批判的吟味寄りなら「**EBM・臨床研究**」、予防・公衆衛生寄りなら「**予防医療・公衆衛生**」。実問題を読んで確定する（1問1カテゴリーなので、どちらか一方に決める）。
- **Q8「整形・骨代謝」**：骨粗鬆症/骨代謝の設問なら「**内分泌・代謝**」、運動器/整形外科の設問なら「**運動器・整形外科**」。実問題を読んで確定する。
- カテゴリー名は確定後は不変扱い。表示ラベルとデータ値を一致させ、変更時は両方を一括で。
- 既存の設計文書（作問グラウンドルール／スキーマ仕様）は `dom`・`cat` 前提で書かれている。本分類の導入後に、それらの記述も追随して更新する（別タスク）。
