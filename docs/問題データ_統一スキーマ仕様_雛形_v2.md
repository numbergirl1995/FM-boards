# GIM Boards 問題データ 統一スキーマ仕様 & 雛形 v2

> **v1からの変更点**：分類を単一化（案A）。旧 `dom`（主領域）＋ `domEn`（英タグ）＋ `cat`（複数タグ配列）を**廃止**し、**単一フィールド `category`** に統一した。`category` の値は「カテゴリー分類 確定仕様 v1」の20カテゴリーのいずれかと**一字一句一致**させる。それ以外の構造はv1を踏襲する。

> サイトは**データ駆動**：全問を `fmboards_data.js`（`const QUESTIONS = [ ... ];`）の配列に「1問 = 1オブジェクト」で格納し、共通レンダラーで描画する。
> 新問追加 ＝ この配列に、検証済みの1オブジェクトを追記する。成果物に氏名は入れない。

---

## 1. トップレベル・スキーマ

| フィールド | 型 | 必須 | ルール・注意 |
|---|---|---|---|
| `category` | string | ✔ | **【新】単一の分類**。「確定仕様v1」の20カテゴリーのいずれかと一字一句一致（例「内分泌・代謝」「感染症」）。**1問1カテゴリー**。旧 `dom`/`domEn`/`cat` は使わない。 |
| `setId` | string | ✔ | セット識別（例 `"set4"`）。5問セットの束ね。分類とは独立。 |
| `title` | string | ✔ | 問題タイトル。 |
| `meta` | string | ✔ | 一行要約（セッティング·年齢性別·要点）。例「外来·65歳男性·HbA1c目標付近だが腎症進行」。 |
| `stem` | string | ✔ | 症例＋設問。**強調（太字）・telegraph・検査値の解釈タグ（（高値）等）は禁止**。 |
| `options` | string[] | ✔ | 選択肢（原則5）。**行為のみ**（「〜を疑い」等の修飾語なし）。正答を最長にしない。 |
| `answer` | integer | ✔ | 正解の**0始まりインデックス**（0=A, 1=B, 2=C, 3=D, 4=E）。**1始まりでない点に注意**。 |
| `explain` | object | ✔ | 解説（下記サブスキーマ）。 |

> ※実データに従来 `id` は無い（v1に記載があったが実装には存在しない）。一意識別は `title`（＋必要なら `setId`）で行う。

## 2. `explain` サブスキーマ（v1から変更なし）

| フィールド | 型 | 必須 | ルール・注意 |
|---|---|---|---|
| `core` | string(HTML) | ✔ | 正解根拠。表・強調可。**§C：症例所見 → 該当する分岐枝 → 数値/正解、の導出を明示する一文を必ず入れる**。 |
| `fig` | string(HTML/SVG) | 任意 | 概念図。**図でしか伝わらないもの限定**。中途半端なら空文字 `""`。 |
| `evidence` | object[] | ✔ | 各文献（`{src,pmid,point,rct,pico?}`）。`rct:true` なら `pico` 6項目必須。 |
| `distractors` | string(HTML) | ✔ | 誤答検討。**各誤答に「なぜ誤りか」＋認知バイアス名**。 |
| `guideline` | string[] | ✔ | 最新ガイドライン（箇条書き）。 |
| `points` | string[] | ✔（**空禁止**） | 習得すべき要点 **4〜5個**（q17水準）。 |
| `refs` | string[] | ✔ | Vancouver形式＋PMID/DOI。**英字の空白を詰めない**。 |

### `evidence[]` の要素構造（v1から変更なし）
```json
{
  "src": "DAPA-CKD,NEJM2020",
  "pmid": "32970396",
  "point": "要点一文",
  "rct": true,
  "pico": { "p": "", "i": "", "c": "", "o": "", "nnt": "", "caveat": "" }
}
```

## 5. 検証チェックリスト（追記前に機械＋目視）

**構造・型**
- [ ] JSONとして妥当（`node --check` 相当が通る）
- [ ] 必須フィールドが全て存在（トップレベル**8**＝`category`/`setId`/`title`/`meta`/`stem`/`options`/`answer`/`explain`＋`explain`内7）
- [ ] `answer` が整数かつ `0 ≤ answer ≤ options数−1`
- [ ] `options` は原則5要素
- [ ] `evidence[]` 各要素に `src`/`pmid`/`point`/`rct`。`rct:true` なら `pico` 6項目
- [ ] `points` が**空でない**（推奨4〜5）

**分類・整合（v2の要点）**
- [ ] **`category` が `CATEGORIES` の20名称のいずれかと一字一句一致**（表記ゆれ・空白・全角半角に注意）
- [ ] 旧 `dom`/`domEn`/`cat` フィールドが残っていない
- [ ] 解答分布（A〜E）が特定文字に偏っていない

**内容・体裁**
- [ ] `pmid` が数字列。**実在＋内容一致を確認**（別工程：事実確認エージェント）
- [ ] `refs` に空白詰まりがない
- [ ] `stem` に強調・解釈タグ・telegraphがない
- [ ] `core` に §C の導出（症例→枝→数値）が明示されている

## 6. 正直な注記

- 本仕様は「カテゴリー分類 確定仕様 v1」に従属する。カテゴリーの名称・数・`imCore` の変更は、まず確定仕様を改訂し、本仕様と実データを追随させる。
- レンダラー側の仕様が変われば、必須フィールドや型も見直す。
