# GIM Boards リポジトリ 作業ガイド（Claude Code用）

## このプロジェクトについて
- 医学試験の問題集サイト。データ駆動：`fmboards_data.js` の `const QUESTIONS = [ ... ];` 配列に「1問=1オブジェクト」で格納し、`fmboards_app.js` が描画する。
- デザインはレンダラー（app.js/styles.css）が担う。**問題ごとにHTMLファイルを作らない。**

## 問題を1問追加するときの厳守ルール
1. **既存の問題（Q1〜）には絶対に触れない。** ファイル全体の作り直し・丸ごと差し替えは禁止。
2. 追加は**配列末尾への1オブジェクト挿入のみ**：最後の問題の閉じ `}` の後ろにカンマを付け、配列を閉じる `];` の直前に新しい `{ ... }` を差し込む。
3. 1オブジェクトの必須キー（トップレベル）：`dom`, `domEn`, `cat`(配列), `setId`, `title`, `meta`, `stem`, `options`(5要素), `answer`, `explain`。
   - ※このデータセットには `id` フィールドは無い。既存の問題に合わせ、新問にも `id` を付けない。
4. `answer` は**0始まりの整数**（0=A, 1=B, 2=C, 3=D, 4=E）。`0 ≤ answer ≤ 4` を満たすこと。
5. `explain` の必須キー：`core`, `fig`, `evidence`, `distractors`, `guideline`, `points`, `refs`。
   - `evidence` は `{src, pmid, point}` 形式のオブジェクト配列。
   - `points` は**空配列にしない**（習得すべき要点を必ず入れる）。
   - `refs` は英単語の空白を詰めない（正: "Author AB, ..." ／ 誤: "AuthorAB,..."）。

## 追加後の検証（コミット前に必ず実行）
- 挿入後、次を報告する：(a) 追加した問題の `id` と `title`、(b) `answer` の値が 0〜4 か、(c) `options` が5要素か、(d) `points` が空でないか、(e) `explain` の7キーが揃っているか。
- **`git diff` を表示し、「変更が"新しい1問の追加"だけで、既存行が消えたり変わっていないこと」を確認する。** 既存行に変更があれば、コミットせず停止して報告する。

## コミットとプッシュ（安全弁）
- **push は、ユーザーが `git diff` を目で見て明示的に許可するまで実行しない。**
- コミットメッセージは英語で `Add question qNN: <短い説明>` の形式にする。
- 成果物にユーザーの氏名を入れない。
